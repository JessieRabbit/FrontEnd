# 0514建立angular開發環境<br/>
## ＠angular專案結構<br/>
Ng new XXX 建立專案 ，也建立angular.cli<br/>
```
* .angular.cil .json => angular設定檔<br/>
* .editorcongfig => 編輯器設定檔，處理斷行符號以及空白...等，＊代表全部檔案<br/>
* .gitignore => git忽略檔案，不需要經歷過版控的檔案與資料夾<br/>
* karma.conf.js => 單元測試工具<br/>
* package.json => 目前專案npm設定檔*（重要）* scripts這個區段，用來定義angular常用的命令<br/>
* protractor.config => m2設定<br/>
* Tsconfig,json => typescript設定<br/>
* Tslint.json => tslint設定檔，用來定義這套工具，程式碼的品質<br/>

* node_modules => npm所有安裝套件<br/>
* e2e  => 與Protractor m2m習習相關，m2m所有測試黨都在此資料夾<br/>
* src => 所有主要結構都在這裡<br/>
* app => 應用程式都放在裡面<br/>
Index.html => 首頁<br/>
Style.css => (global style) 整個應用程式，會用到此css<br/>
main.ts => angular javascript進入點 ，執行完之後，就直接到app底下 app.module<br/>
platformBrowserDynamic().bootstrapModule(AppModule)<br/>
  .catch(err => console.log(err));<br/>
appComponent => 根元件<br/>

* assets => 所有靜態檔案都放在這 .gitkeep 純粹可以進入git 版控的東西<br/>
* environments => angular應用程式會用到的環境變數，prod 內容在bulid的時候才會使用，開發只會用到不是prod的environments<br/>

* polyfills.ts => angular在執行的時候，在舊的劉覽器，會導致程式碼無法生效，這時可以把需要的版本劉覽器打開，就可以生效<br/>

* Test.ts => 單元測試也會用到，被kama使用<br/>
* Tsconfig.app .json => 繼承上一個Tsconfig,json，並且作出額外定義，主要更改為types這個區塊<br/>
* Tsconfig.spec .json => 單元測試typescript<br/>
* Typings.d.ts => 額外typescript型別定義<br/>
```
